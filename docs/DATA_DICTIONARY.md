# データ辞書 — 匿名化方針・用語対照表

## 匿名化宣言

本プロジェクトで使用するすべてのデータは合成データ(ダミーデータ)である。`荷主A`〜`荷主F`(主要荷主)・`ロングテール(集約)` はランダムに割り当てられた匿名ラベルであり、特定の実在荷主・取引先を指さない。伝票種別(電子伝票/自社手書き伝票/荷主仕様伝票/代理店伝票)は業務上一般的なカテゴリ名であり、特定企業の実測構成比ではなく「業界的にありうる構成の一例」として提示している。全国日次件数(1万件/日)・登録荷主数レンジ(1〜60社)を含め、コスト・単価・信頼度スコア等の数値はすべて架空の前提値から `mulberry32` シード付き疑似乱数で生成しており、実際のコスト実績とは意図的に一致させていない。詳細な設計方針は [design.md](design.md) の「1. 匿名化・安全設計(非交渉事項)」を参照。

## 用語対照表(フィールド名 ⇔ 表示名)

CSV/JSONのキー・変数名は英語のまま維持し(スキーマの安定性のため)、画面表示・グラフ・README等の人が読む部分だけを日本語化する二層方式を採る。

### 伝票・OCR抽出結果

| フィールド名(英語) | 表示名(日本語) | 説明 |
|---|---|---|
| `slip_id` | 伝票ID | 匿名連番ID(例: `SL-000123`) |
| `slip_type` | 伝票種別 | `handwritten`(自社手書き伝票) / `shipper_format`(荷主仕様伝票) / `agency_format`(代理店伝票) |
| `shipper_id` | 荷主ID | `荷主A`〜`荷主F`(主要荷主)または `ロングテール(集約)` |
| `extracted_weight_kg` | OCR抽出重量(kg) | 合成値 |
| `confidence` | OCR信頼度スコア | 0〜100の合成値 |
| `has_charter` | チャーター有無 | 合成値 |
| `has_incidental_work` | 付帯作業有無 | 合成値 |
| `rate_basis` | 運賃適用区分 | `blanket`(基本包括: 顧客と合意済みの包括運賃、件数の大半) / `special`(特別適用: 案件ごとの個別見積、少数だが訂正工数が集中)。ロングテール荷主ほど `special` の比率が高い(架空) |
| `status` | 処理ステータス | `auto_approved`(自動承認) / `needs_review`(要人手確認) |
| `transport_mode` | 輸送方法 | `air_first` / `air_day` / `air_last`(航空便: 初便/昼間便/最終便) / `truck_route`(トラック便: ルート別) |

### 荷主別ルールマスター(二層レート)

ルールエンジンは Layer1(基本包括レートマスター)+ Layer2(特別適用オーバーライド)の二層で構成する。詳細は [design.md](design.md) 3章、[strategy_options.md](strategy_options.md) 3章を参照。

| フィールド名(英語) | 表示名(日本語) | 説明 |
|---|---|---|
| `shipper_id` | 荷主ID | `荷主A`〜`荷主F`(主要荷主)または `ロングテール(集約)` |
| `blanket_rate` | 基本包括レート(Layer1) | 顧客と合意済みの包括運賃(架空)。現状は基幹に無く人手で再現している部分 |
| `weight_tier_rates` | 重量帯別単価テーブル | 合成値 |
| `incidental_rate` | 付帯作業単価 | 合成値 |
| `charter_condition` | チャーター適用条件 | 合成値・簡略化した条件文言 |
| `special_override` | 特別適用オーバーライド(Layer2) | 案件ごとの個別見積・例外を Layer1 への差分として適用(架空)。C(標準化)が効くのはこの層 |

### コストシミュレーション

| フィールド名(英語) | 表示名(日本語) | 説明 |
|---|---|---|
| `daily_slip_volume_national` | 全国日次伝票件数 | 架空値。1万件/日を想定 |
| `shipper_count` | 登録荷主数 | シミュレーション用パラメータ。1〜60社のレンジ(架空、全社規模を想定) |
| `monthly_slip_volume` | 月間伝票件数 | シミュレーション用パラメータ(架空) |
| `auto_approval_rate` | 自動承認率 | OCR信頼度閾値に応じて変動 |
| `hours_without_engine` | ルールエンジンなしの月間運用工数(架空) | 荷主数 n の関数 `4n + 0.15n(n-1)`。荷主間の例外対応が組み合わせ的に積み上がることを表現 |
| `hours_with_engine` | ルールエンジンありの月間運用工数(架空) | 荷主数 n の関数 `8 + n`。固定オンボーディング+荷主ごとのマスター登録のみ |

### 経営オプション比較(payback / NPV モデル)

すべて架空の前提値。詳細は [strategy_options.md](strategy_options.md) 4章を参照。

| フィールド名(英語) | 表示名(日本語) | 説明 |
|---|---|---|
| `baseline_monthly_cost` | 月間訂正コストのベースライン(基準) | 架空値 240万円/月(電子化率60%・集約なしの現状)。全国日次1万件×OCR対象40%×手作業比率×みなし単価から組み立て、実績とは無関係 |
| `eslip_rate` | 電子化率(Dスライダー) | e-伝票比率 60〜90%(架空)。母数を縮小: ベースライン係数 = (100−電子化率)/40 |
| `consolidation_reduction` | 集約による単価低減(Eスライダー) | 0〜30%(架空)。残存手作業の単価を低減: ベースライン係数 = (1−集約削減率) |
| `effective_baseline` | 実効ベースライン | D/E 反映後の月間訂正コスト = 240 ×(100−電子化率)/40 ×(1−集約削減率)。A/B/C の削減額はこの実効ベースラインに乗る |
| `capex` | 一時投資 | オプション別の初期投資(架空)。A=320, B=3,000(スライダー1,000〜8,000), C=180 万円 |
| `opex` | 月次運用費 | オプション別の運用費(架空)。A=12, B=20, C=0 万円/月 |
| `reduction_rate` | 月間削減率 | ベースラインに対する削減率(架空)。A=45%, B=80%, C=可変0〜60% |
| `ramp_months` | 効果の立ち上がり月数 | 削減効果が線形に立ち上がる月数(架空)。A=2, B=9, C=6 |
| `payback_month` | 回収期間 | 累計純便益が初めて0を上回る月 |
| `npv` | 36ヶ月NPV | 割引率0.5%/月・評価期間36ヶ月の正味現在価値(架空) |
| `effective_capex_b` | Bの実効投資額 | Cの標準化削減率で圧縮後のBの投資額。`capex_B × (1 − 0.4 × std_c)`(圧縮率0.4は架空、順序効果の表現) |
